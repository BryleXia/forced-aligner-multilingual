● 你今天做的事：语音语料库的强制对齐

  ---
  任务定性

  你做的是 Forced 
  Alignment（强制对齐），这是语料库语言学和语音学研究的核心预处理步骤。

  具体来说：给定一段语音录音和对应的文字转写稿，计算出每个句子（乃至每个词）在录
  音中的精确起止时间，生成带时间戳的字幕文件（SRT 格式）。

  ---
  技术栈

  WhisperX
  OpenAI Whisper 的增强版框架。原版 Whisper
  只能输出句子级时间戳，精度较粗。WhisperX
  在此基础上集成了强制对齐模块，可将时间戳精细到词级别。

  faster-whisper large-v3
  底层 ASR（Automatic Speech Recognition，自动语音识别）模型。参数量约 15
  亿，是目前开源 ASR 模型中精度最高的量级之一。使用 CTranslate2 框架推理，比原版
   Whisper 快约 4 倍。你本地已有该模型（2.9GB）。

  wav2vec2
  Meta AI 开发的自监督语音表示模型，此处用于 强制对齐
  的精细化阶段。它以帧为单位（约每秒 50
  帧）分析音频，将文字序列与声学特征精确对齐。我们用的是西语专项版本
  wav2vec2_voxpopuli，在西班牙语语料上训练。

  CUDA / GPU 加速
  模型推理在 GPU 上并行运算。你本地 RTX 3060（6GB 显存），服务器 RTX 5090（32GB
  显存）。显存越大，能处理的音频越长，速度越快。

  ---
  流程原理

  原始音频 (.m4a)
        │
        ▼
  ① VAD（Voice Activity Detection）
    用 pyannote 检测音频中的有声/静音段
    过滤掉背景噪声，确定语音区间
        │
        ▼
  ② ASR 转录（faster-whisper large-v3）
    将音频转为文字，同时输出句子级粗时间戳
    → 告诉我们"哪段话大概在什么时间"
        │
        ▼
  ③ 强制对齐（wav2vec2）
    在②的时间窗口内，将文字精确对齐到音频帧
    → 输出词级精确时间戳（精度约 20ms）
        │
        ▼
  ④ 文字映射
    ③给的时间戳来自转录文字，但你的 SRT 文字才是权威
    → 按词数比例，将原始 SRT 每行文字对应到③的时间戳
        │
        ▼
  输出：原始文字 + 音频推导的精准时间戳 (.aligned.srt)

  ---
  为什么要两个模型？

  这是文科生最容易困惑的地方：

  - large-v3 的能力是"听声音，猜文字"——它不看你的 SRT，它自己转录
  - wav2vec2 的能力是"在已知时间窗口内，把文字精确贴到音频上"

  wav2vec2 本身不知道时间，它需要别人告诉它"这句话大概在第 30 秒到第 40
  秒之间"，它才能在这个窗口里精确定位每个词。

  你的原始 SRT 时间戳是乱写的，不能用。所以我们先让 large-v3
  转录一遍音频，得到可信的时间窗口，再把这个窗口交给 wav2vec2
  精细对齐，最后再把你的原始文字覆盖上去。

  你做了哪些操作，每步在干什么

  pip install whisperx
  pip 是 Python 的包管理器。这条命令让服务器去网上下载 WhisperX

  apt-get 是 Ubuntu 系统的软件安装工具，相当于手机上的应用商店。ffmpeg
  是一个音视频处理工具，WhisperX
  依赖它来读取音频文件。这次没装成功，因为镜像里的应用商店没有收录它，最后用了
  imageio-ffmpeg 自带的版本代替。

  ln -s 是创建"快捷方式"（Linux 叫软链接）。imageio-ffmpeg 带了一个 ffmpeg
  程序，但名字很长（ffmpeg-win-x86_64-v7.1），系统找不到它。这条命令在系统的公共
  路径下创建了一个叫 ffmpeg 的快捷方式指向它，系统就能找到了。

  sed -i 's|旧路径|新路径|g' align_srt.py
  sed 是 Linux 的文本替换工具。因为脚本里写的是你本地 Windows
  的路径（E:/my-ai-studio/...），服务器上根本没有 E
  盘，所以要把路径换成服务器上的路径（/root/...）。这条命令相当于批量查找替换。

  python align_srt.py
  让服务器运行你的 Python 脚本，开始干活。

  zip -r aligned.zip aligned/
  把 aligned 文件夹打包成 zip 压缩文件，方便从 JupyterLab 下载回本地。

  ---
  文件系统的逻辑

  Linux 没有 C 盘 D 盘的概念，用的是树状路径：

  /                     ← 根目录（相当于整个系统）
  ├── root/             ← 你登录后的"家目录"，相当于桌面
  │   ├── align_srt.py  ← 你上传的脚本
  │   ├── *.m4a         ← 你上传的音频
  │   ├── aligned/      ← 脚本输出的结果
  │   └── autodl-tmp/   ← 数据盘（快，但不随镜像保存）
  │       └── huggingface/  ← large-v3 下载到这里
  └── usr/local/bin/    ← 系统公共命令存放处（ffmpeg 快捷方式放这里）

● 对，就是它。

  Linux 其实没有什么神秘的，本质上就是一个操作系统，和 Windows 一样，只是：

  - 没有图形界面（默认），所有操作靠打命令
  - 路径用斜杠，没有盘符，从 / 开始
  - 大小写敏感，align_srt.py 和 Align_srt.py 是两个不同的文件
  - 用户是 root，相当于 Windows 的管理员，权限极大，没有任何拦截提示

  你今天用到的命令其实就那几类：

  ┌─────────────────┬───────────────┬────────────────────────┐
  │      命令       │    干啥的     │      Windows 类比      │
  ├─────────────────┼───────────────┼────────────────────────┤
  │ ls              │ 列出文件      │ dir                    │
  ├─────────────────┼───────────────┼────────────────────────┤
  │ cd              │ 进入目录      │ 双击文件夹             │
  ├─────────────────┼───────────────┼────────────────────────┤
  │ cp / mv         │ 复制/移动文件 │ Ctrl+C/X → Ctrl+V      │
  ├─────────────────┼───────────────┼────────────────────────┤
  │ rm              │ 删除文件      │ 删除键（没有回收站！） │
  ├─────────────────┼───────────────┼────────────────────────┤
  │ pip install     │ 装 Python 包  │ 应用商店               │
  ├─────────────────┼───────────────┼────────────────────────┤
  │ apt-get install │ 装系统软件    │ 应用商店               │
  ├─────────────────┼───────────────┼────────────────────────┤
  │ python xxx.py   │ 运行脚本      │ 双击 .py 文件          │
  └─────────────────┴───────────────┴────────────────────────┘

  世界上大多数服务器、超算、手机（Android 底层）、路由器跑的都是
  Linux。你今天算是正式入门了。
