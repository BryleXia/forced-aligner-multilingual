  ---
  8号实验做了什么改动

  在 align_srt.py 的 boundary_extend() 函数里，三项修复：

  1. 只用 prefix_words[0] 匹配句首词（之前用多个词，导致常见词 la
  误命中错误位置）
  2. 前两词拼接兜底：prefix_words[0] + prefix_words[1] 处理 ASR 合并词（如
  laobao 能匹配 laopao）
  3. fuzzy_thr 从 0.40 提至 0.45（覆盖 nuodeng→nodon 这类拼写偏差）

  效果：seg001-003 共 237 句，瑕疵率 3.8%（9句），主要问题是早结束/晚进，偏差
  0.5~2 秒。已打 git tag v8-milestone，随时可 git checkout v8-milestone 退回。

  ---
  调研出来的三条重构路线

  路线A（最推荐）：绕过 Whisper ASR，直接把 SRT 文本喂给 whisperx.align()
  - 格式：[{"text": "...", "start": 0.0, "end": 5.0}]
  - 彻底消灭 LCS + boundary_extend 整套逻辑
  - 可行性已通过社区 issue #939/#1308 确认

  路线B：WhisperX 粗对齐 + MFA（Montreal Forced Aligner）精对齐
  - MFA v3.1.2 西语声学模型，词级准确率 41.6%（WhisperX 仅 22.4%）
  - 缺点：~20个中文罗马音专名需要手写发音条目

  路线C：ctc_forced_aligner（deskpai/ctc_forced_aligner）
  - 原生支持 SRT 输出，显式支持西语，pip install ctc-forced-aligner
  - 最省事，但效果未经验证

  ---
  OOV 问题根本原因

  WhisperX 的 wildcard 机制：wav2vec2 词表外的字符被替换为
  *，时间戳来自"最大非空白概率"，对陌生音素序列（中文罗马音专名）不可靠。路线A
  能绕过这个问题。
