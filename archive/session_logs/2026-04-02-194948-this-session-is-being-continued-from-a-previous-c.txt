  根本原因诊断

  触发点：第15行 end_idx 多走了1步

  第15行是全文最长的句子（~32个词）。词数估算走到末尾时，end_idx
  指向的单词恰好在 03:10.5 秒附近——而 "El aroma" 的真实位置就在 03:10,502。

  audio_pos = end_idx + 1 执行后，audio_pos 刚好越过了 "el" 这个词。

  第16行锚定失败

  第16行搜索从 audio_pos（大约03:11处）开始往前找 window=150 个词。但 "el aroma"
   在 03:10 处——在 audio_pos 前面一点点，被跳过了。window=150
  再大也没用，因为搜索只往后走，不往前回头。

  结果锚定到了之后出现的假匹配 "el..." 在 03:17,949。

  雪崩：第18行突跳49秒

  第16行锚点晚了7秒 → audio_pos 整体前移7秒 → 第17行也晚7秒 → audio_pos
  继续累积。到第18行时，audio_pos 已越过 "El pueblo
  Gidi"（03:37）约8秒，落在03:45附近。window=150
  向后搜，找到的最早假匹配是04:26——跳了49秒。此后一路雪崩。

  ---
  可能的解决方案

  方案A（推荐）：允许小量回望（lookback）

  把锚定搜索的起点从 audio_pos 改为 max(0, audio_pos - 15)，window 相应调整。

  - 精准对症：专门针对"多走了几步"的场景
  - 只回望15个词（约5秒），不会回到上上一句造成误匹配
  - 实现改动最小，只改find_line_anchor的调用方式

  方案B：保守推进 audio_pos（减半）

  把 audio_pos = end_idx + 1 改为 audio_pos = anchor + max(1, len(srt_wn) // 
  2)，只推进词数的一半。

  - 0号实验之所以正确，很可能是因为某个历史版本的 audio_pos 推进较保守
  - 副作用：对于紧密连续的句子，推进不够，下一行会在更靠前的窗口里搜，存在误匹配
  早出的风险

  方案C（同时使用A+B）

  两个改动一起上，效果叠加——最保险，但改的东西多一点。

  ---
  你想用哪个方案？
  我倾向于推荐方案A+B组合，但如果你想最小改动先验证，可以先试方案A。

     def match_srt_to_words(srt_lines, words):
         if not words:
             return [{"start": 0, "end": 0, "text": line} for line in srt_lines]

         audio_norm = [normalize_word(w["word"]) for w in words]
         audio_pos = 0
         result = []
         n = len(srt_lines)

         for i, line in enumerate(srt_lines):
             srt_wn = normalize(line)
             if not srt_wn:
                 prev_end = result[-1]["end"] if result else 0
                 result.append({"start": prev_end, "end": prev_end + 0.5,
     "text": line})
                 continue

             remaining = n - i - 1
             audio_pos = max(0, min(audio_pos, len(words) - remaining - 1))

             # 关键修复①：允许往回看15个词，防止end_idx多走1步时跳过正确锚点
             search_start = max(0, audio_pos - 15)
             anchor = find_line_anchor(srt_wn, audio_norm, search_start,
     window=165)
             anchor = min(anchor, len(words) - 1)
             end_idx = min(anchor + len(srt_wn) - 1, len(words) - 1)

             seg_start = words[anchor]["start"]
             seg_end = words[end_idx]["end"]

             result.append({"start": seg_start, "end": seg_end, "text": line})
             # 关键修复②：保守推进，只推进一半词数，防止长句overshoot
             audio_pos = anchor + max(1, len(srt_wn) // 2)

         # 事后钳制：强制每行在下一行开始前结束，消除停顿期滞留
         for i in range(len(result) - 1):
             next_start = result[i + 1]["start"]
             if result[i]["end"] > next_start - 0.05:
                 result[i]["end"] = max(next_start - 0.05, result[i]["start"] +
     0.1)

         return result
