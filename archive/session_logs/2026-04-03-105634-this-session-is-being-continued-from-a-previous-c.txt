❯ 如果上MFA，一切都会好吗？                                                     

● 不会自动好。

  MFA的核心优势是发音词典——它按音素对齐，不靠ASR转录。对标准西语词，精度比Whispe
  rX高。

  但你的问题恰好卡在词典上：老包/诺邓/香格里拉/卓玛 这些词不在任何西语词典里。MF
  A遇到OOV（out-of-vocabulary）词有两种结果：要么跳过，要么崩掉对齐。

  能好的前提：你手动给每个专有名词写发音条目（用IPA或ARPAbet），加进词典。做了这
  一步，MFA的边界精度会显著优于现在的模糊扩展方案。

  所以答案是：做了词典维护，会好很多；不做，可能更糟。

❯ 不能两种混搭吗？                                                              

● 可以，而且这是最合理的方案：

  - 普通西语词 → MFA处理，音素级精度
  - 中文音译专有名词 →

  ---
  调查结论

  你遇到的问题有了学术解释

  WhisperX的alignment.py里有个wildcard机制：wav2vec2词汇表里没有的字符（比如"zh"
  、"uo"等音译名），会被替换成wildcard 
  token，取的是该帧最高的non-blank概率——相当于"在任意帧都可能高置信度命中"。这就
  是为什么"Zhuoma"、"Nuodeng"的边界时间戳乱跳的根本原因。

  ---
  关键发现：三条出路

  出路1（最快见效，无需改架构）
  在把SRT文本送进whisperx.align()之前，把音译词预处理成wav2vec2认识的形式：
  - Zhuoma → chuoma
  - Nuodeng → nuodeng（可能已有效）
  - Shangri-La → shangrila

  直接绕过wildcard问题，代价是维护一个映射表，一次性工作。

  出路2（精度最高，需要一次性配置）
  切换到MFA做强制对齐：WhisperX只做粗粒度分段，MFA用信任的SRT文本做精确词级对齐
  。需要给~20个中文音译名手写发音条目（30分钟工作），之后永久有效。Interspeech
  2024论文实测：MFA词级精度41.6% vs WhisperX 22.4%（10ms容差）。

  出路3（最省事，现成SRT输出）
  试一下 https://github.com/deskpai/ctc_forced_aligner，明确支持西语，原生SRT输
  出，18个对齐模型可选，pip install ctc-forced-aligner就能用。

  ---
  我的建议

  先试出路1——五分钟改动，不换架构，验证效果。如果音译名的wildcard问题是主要瓶颈
  ，这个最快。
  长期来看**出路2（MFA）**才是真正的解法，特别是如果你以后处理更多这类素材。

     Context：为什么要换架构

     历次实验（0-4号）都在给同一个贪心算法打补丁。根本问题是：

     我们在用逐行贪心搜索模拟全局序列对齐，这两件事不等价。

     贪心的致命缺陷：第k行的错误（audio_pos
     走偏）会传染第k+1、k+2行，无法回头修正。
     再加上两个结构性设计雷点：
     1. end_idx = anchor + len(srt_wn) - 1——用词数猜结束位置，一旦ASR漏词/专名拆
     分就失真，且失真会通过audio_pos放大传播

     - 一次全局求解，绝无级联错误
     - 单调性由DP路径保证（时间不倒退）
     - 行的结束时间来自真实匹配到的最后一个token，不再靠词数估算
     - 未匹配行（专名等ASR无法覆盖的）做邻近插值，不强行硬匹配

     def normalize_for_dp(text):
         """规范化文本为词序列，连字符拆为空格（修复专名拆分失配）"""
         text = text.lower()
         text = text.replace("-", " ")
         text = unicodedata.normalize("NFD", text)
         text = "".join(c for c in text if unicodedata.category(c) != "Mn")
         text = re.sub(r"[^\w\s]", "", text)
         return text.split()

     2. 新增 dp_align 函数

     LCS全局对齐，返回匹配的 (srt_token_idx, asr_token_idx) 对：

     def dp_align(srt_seq, asr_seq):
         """
         LCS全局对齐（gap penalty=0，只统计完全匹配）。
         srt_seq: [(norm_word, line_idx), ...]
         asr_seq: [(norm_word, word_idx), ...]   ← word_idx 指向 words[] 中的词
         返回: [(srt_token_idx, asr_token_idx), ...] 匹配对列表（已保证单调）
         """
         N, M = len(srt_seq), len(asr_seq)
         dp = [[0] * (M + 1) for _ in range(N + 1)]
         for i in range(1, N + 1):
             for j in range(1, M + 1):
                 if srt_seq[i-1][0] == asr_seq[j-1][0]:
                     dp[i][j] = dp[i-1][j-1] + 1
                 else:
                     dp[i][j] = max(dp[i-1][j], dp[i][j-1])
         # 回溯
         pairs = []
         i, j = N, M
         while i > 0 and j > 0:
             if srt_seq[i-1][0] == asr_seq[j-1][0] and dp[i][j] == dp[i-1][j-1]
     + 1:
                 pairs.append((i-1, j-1))
                 i -= 1; j -= 1
             elif dp[i-1][j] >= dp[i][j-1]:
                 i -= 1
             else:
                 j -= 1
         pairs.reverse()
         return pairs

     3. 新增 match_srt_to_words_dp（替换原有函数）

     def match_srt_to_words_dp(srt_lines, words):
         if not words:
             return [{"start": 0, "end": 0, "text": line} for line in srt_lines]

         # 展开SRT：每个词带行号
         srt_seq = []
         for i, line in enumerate(srt_lines):
             for w in normalize_for_dp(line):
                 srt_seq.append((w, i))

         # 展开ASR：每个词带原始words[]序号（连字符同样拆分）
         asr_seq = []
         for j, w in enumerate(words):
             for part in normalize_for_dp(w["word"]):
                 asr_seq.append((part, j))

         # 全局DP对齐
         pairs = dp_align(srt_seq, asr_seq)

         # 从匹配对中提取每行的首尾ASR词
         line_first = {}   # line_idx → asr word_idx（最早匹配）
         line_last  = {}   # line_idx → asr word_idx（最晚匹配）
         for srt_i, asr_i in pairs:
             line_idx     = srt_seq[srt_i][1]
             asr_word_idx = asr_seq[asr_i][1]
             if line_idx not in line_first:
                 line_first[line_idx] = asr_word_idx
             line_last[line_idx] = asr_word_idx

         # 构建结果，未匹配行插值
         result = []
         for i, line in enumerate(srt_lines):
             if i in line_first:
                 seg_start = words[line_first[i]]["start"]
                 seg_end   = words[line_last[i]]["end"]
                 result.append({"start": seg_start, "end": seg_end, "text":
     line})
             else:
                 prev_end = result[-1]["end"] if result else 0
                 result.append({"start": prev_end, "end": prev_end + 0.5,
     "text": line})

         # 事后钳制（保留）
         for i in range(len(result) - 1):
             next_start = result[i + 1]["start"]
             if result[i]["end"] > next_start - 0.05:
                 result[i]["end"] = max(next_start - 0.05, result[i]["start"] +
     0.1)

         return result
