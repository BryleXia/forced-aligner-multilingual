# 西语字幕强制对齐工具

> 一个学生项目。把西班牙语字幕文件的时间戳对齐到音频上，在测试集上做到了**一句不差**。

**作者：** BryleXia · 北京第二外国语学院欧洲学院  
**测试结果：** 5段音频 / 390条字幕 / 约56分钟 · 人工逐句审听 · **0条错误**

---

## 这个工具做什么

有时候我们拿到一个视频，配套的字幕文件文字是对的，但时间戳不准——字幕出来得早了或晚了，和说话对不上。

这个工具就是解决这个问题的：**把字幕文字和音频对齐，重新生成准确的时间戳**。

---

## 为什么不用现成的工具

市面上有 WhisperX、MFA 等工具可以做强制对齐，但它们的标准流程是：

```
音频 → 先用 AI 把音频转录成文字 → 再把转录的文字和字幕文字做匹配
```

这对普通内容没什么问题。但我们的字幕里有大量**中文人名、地名的西班牙语音译**，比如：

| 正确写法 | AI 转录出来的 |
|---------|------------|
| Lao Bao（老包） | "Alien"、"Lab" |
| Nuodeng（诺邓） | "Nudon"、"Nodeng" |
| Shengwu（生五） | "Chengwu" |
| A Liang（阿亮） | "Alien"（完全不认识） |

AI 没见过这些词，就乱猜。一旦转录出来的文字和字幕文字对不上，时间戳就跑偏了。

---

## 我们的做法

既然字幕文字本来就是对的，**为什么要先转录再匹配？直接跳过转录那一步不就好了吗？**

这个想法虽然简单，但确实管用：

```
标准做法：  音频 → [AI转录] → 有错误的文字 → [对齐] → 时间戳
                                    ↑ 专名在这里出问题

我们的做法：音频 ─────────────────────────→ [对齐] ← 字幕文字 → 时间戳
```

直接把字幕文字喂给对齐器（wav2vec2），让它在音频里找每个词的位置。因为输入的文字本来就是正确的，对齐成功率极高，不再需要任何专名词典。

---

## 还解决了一个隐藏问题

用这个做法跑起来之后，发现一个新问题：遇到场景切换（长时间静音）的地方，对齐器有时会把下一句字幕的起点放得过早——早个几秒甚至十几秒。

原因是：对齐器必须把每个字幕词都安排到音频的某一帧，它不会自动跳过静音段，就只好把词往前堆。

我们用了一个小方法来检测和修正这类错误：**如果一句字幕的"词速"异常低**（正常说话2-4词/秒，被拉伸后不到1词/秒），而且持续时间比较长，而且和前一句几乎没有间隔——这三个条件同时满足，说明这句大概率出了问题，就去音频里找真正的语音起点，把时间戳挪过去。

这个方法在我们的测试集上触发了3次，全部修正正确，没有一次误判。

修正案例：

| 位置 | 修正前 | 修正后 | 早了多久 |
|------|--------|--------|---------|
| 第3段第16句 | 02:00 | 02:17 | 17秒 |
| 第3段第37句 | 04:44 | 04:51 | 7秒 |
| 第5段第6句 | 00:39 | 00:45 | 6秒 |

---

## 测试结果

| 方案 | 测试句数 | 出错句数 | 错误率 |
|------|---------|---------|--------|
| 旧方案（先转录再匹配） | 237 | 9 | 3.8% |
| 本工具（跳过转录） | **390** | **0** | **0%** |

旧方案里那几个死活修不好的专名（"A Liang"被识别成"Alien"等），在新方案里自然消失了——因为我们根本不做转录。

## 使用方法

### 安装依赖

```bash
pip install whisperx silero-vad num2words imageio-ffmpeg
```

### 准备文件

把音频和字幕放在同一个目录，文件名保持一致（只有扩展名不同）：

```
/root/
├── seg001.m4a
├── seg001.asr.qc.srt
├── seg002.m4a
├── seg002.asr.qc.srt
└── ...
```

### 运行

```bash
python align_srt_routeA.py
```

对齐后的字幕保存在 `/root/aligned_routeA/` 目录下。

### 修改路径

打开脚本，修改开头几行：

```python
AUDIO_DIR  = Path("/root")                   # 音频和字幕所在目录
OUTPUT_DIR = Path("/root/aligned_routeA")    # 输出目录
DEVICE     = "cuda"                          # 有 GPU 用 "cuda"，没有用 "cpu"
LANGUAGE   = "es"                            # 语言代码，西语是 "es"
```

---

## 局限性

- 字幕文字必须是对的。如果字幕本身有错字、漏句，工具无法自动纠正。
- 单段音频超过30分钟时会自动分块处理，分块边界处可能有轻微误差。
- 目前只在西班牙语上完整测试过。

## 参考

这个工具基于 [WhisperX](https://github.com/m-bain/whisperX)（Bain et al., Interspeech 2023）和 [Silero-VAD](https://github.com/snakers4/silero-vad)，wav2vec2 对齐模型来自 [Baevski et al., NeurIPS 2020](https://arxiv.org/abs/2006.11477)。

与 MFA（Montreal Forced Aligner）的精度比较数据来自 Rousso et al.（Interspeech 2024，[arXiv:2406.19363](https://arxiv.org/abs/2406.19363)）。
